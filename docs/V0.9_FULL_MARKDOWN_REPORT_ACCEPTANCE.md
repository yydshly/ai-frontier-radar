# V0.9 完整 Markdown 报告导出验收

## 目标

把一张 InsightCard 导出为完整的中英双语资料编译报告，用于长期知识沉淀和复盘。

## 和行动任务导出的区别

| 维度 | V0.5 行动任务导出 | V0.9 完整报告导出 |
|------|-------------------|-------------------|
| 目标 | 继续拆解执行 | 知识沉淀、长期复盘 |
| 内容 | 行动建议、产品机会 | 原文保真、中英双语、洞察全量 |
| 用户场景 | 下一步做什么 | 这篇资料对我意味着什么 |

## 报告结构

完整报告包含以下章节：

1. **原文信息**：标题、链接、作者、时间、模型、相关性分数、处理状态
2. **English Core Summary**：英文核心摘要
3. **Original Key Claims**：原文主要主张（英文）
4. **Key Evidence Points**：英文证据点
5. **Key Terms EN-ZH**：术语中英对照表
6. **中文解说**：中文解说（来自双语报告）
7. **中文摘要**：中文摘要（来自 InsightCard）
8. **关键事实**：关键事实列表
9. **技术洞察**：技术洞察列表
10. **产品机会**：产品机会列表
11. **风险与注意事项**：风险列表
12. **行动建议**：行动建议列表
13. **相关性判断**：相关性分数、匹配方向、判断理由
14. **用户判断**：当前判断和用户备注
15. **保真提示**：保真说明
16. **解读边界**：解读边界说明
17. **后续可继续追问的问题**：4 个引导问题

## 验收命令

### 有双语报告的验收

```bash
python scripts/acceptance_export_full_report.py --isolated-db --with-bilingual
```

验证：
- 报告包含英文核心摘要、原文主张、证据点、术语对照、中文解说
- 报告包含保真提示和解读边界
- 下载文件名为 `insightcard-{id}-report.md`

### 无双语报告的验收

```bash
python scripts/acceptance_export_full_report.py --isolated-db --without-bilingual
```

验证：
- 报告不崩溃
- 显示"暂无中英双语报告"
- 仍然包含中文摘要、关键事实、技术洞察等 InsightCard 字段

## 手动验收

### 步骤 1：准备数据

1. 启动服务：`uvicorn app.main:app --reload --port 8779`
2. 打开 `http://127.0.0.1:8779/`
3. 提交一个英文 AI 前沿文章 URL，编译为 InsightCard
4. 在卡片详情页点击「生成中英双语报告」

### 步骤 2：导出完整报告

1. 打开 `http://127.0.0.1:8779/cards/{id}`
2. 在「看完后的判断」区域找到「📄 导出完整 Markdown 报告」按钮
3. 点击进入报告预览页
4. 确认预览包含所有章节
5. 点击「下载完整 Markdown 报告」
6. 确认下载文件名为 `insightcard-{id}-report.md`

### 步骤 3：从列表导出

1. 打开 `http://127.0.0.1:8779/cards`
2. 在任意卡片的操作列找到「完整报告」链接
3. 点击直接进入报告预览

## 通过标准

1. 有双语报告时，报告包含英文核心摘要、原文主张、证据点、术语表、中文解说
2. 无双语报告时，报告不崩溃，并提示"暂无中英双语报告"
3. 下载文件名格式为 `insightcard-{id}-report.md`
4. 预览页面和下载路由都可用
5. 不影响行动任务导出（V0.5）

## 不影响的功能

- `/cards/{id}/export-markdown` 行动任务导出仍然可用
- `/cards/{id}/export-markdown/download` 仍然可用
- 双语报告生成仍然可用
