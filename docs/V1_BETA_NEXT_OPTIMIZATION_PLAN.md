# V1.0-beta Next Optimization Plan

> 规划日期：2026-06-11
> 当前阶段：V1.0-beta.22 — First Usable Loop 已验收通过

---

## 1. 当前状态

V1.0-beta First Usable Loop（首个可日常使用的闭环）已验收通过。

用户主链路已跑通：

```
进入今日雷达 → 更新今日新增 → 生成中文摘要 → 查看推荐深入分析
→ 生成/查看洞察卡 → 查看今日报告 → 生成今日核心报告
```

但这只是"链路跑通"，还不是"日常好用"。

---

## 2. 下一阶段目标

**从"链路跑通"进入"日常可用覆盖率提升"。**

核心指标变化：

| 指标 | 当前 | 下一阶段目标 |
|------|------|-------------|
| 中文摘要覆盖率 | ~1% | 今日页 Top 20 稳定中文化 |
| 洞察卡生成 | 全量散乱 | 推荐 Top 3-5 优先生成 |
| 今日报告 | 规则版可用 | 规则版输出有价值 |
| 正文抓取 | 按需可抓 | 确认抓取成功率 |

---

## 3. 中文摘要覆盖率提升

### 3.1 当前问题

- 大部分 SourceItem 没有中文摘要
- 新增内容需要用户手动触发摘要生成
- 用户不知道哪些内容缺摘要

### 3.2 下一阶段目标

**今日页 Top 20 稳定中文化**（非全量）：

1. 每日首次访问今日雷达时，自动对 Top 20 缺摘要内容生成中文摘要
2. 摘要生成上限受 `ONE_LINER_MAX_PER_RUN=20` 保护
3. 已有中文摘要的内容自动跳过
4. 不在首次访问时全量生成 — 渐进补全

### 3.3 后续目标

**每日新增 Top 50 渐进补全**：

- 每日新抓取的内容，按推荐优先级渐进生成摘要
- 不做一次性全量生成（成本高，风险大）
- 优先对"推荐深入分析"候选生成摘要

---

## 4. 洞察卡生成策略

### 4.1 当前问题

- 用户可以手动生成任意内容的洞察卡
- 没有优先级引导，容易生成无价值的洞察卡
- 全量生成成本高

### 4.2 下一阶段目标

**推荐深入分析 Top 3-5 优先生成洞察卡**：

1. 在"推荐深入分析"区直接提供"生成洞察卡"按钮
2. 只对 Top 3-5 推荐候选显示"生成洞察卡"快捷按钮
3. 其他内容仍可通过详情页手动生成

### 4.3 后续目标

- 已有洞察卡的内容在推荐区显示"查看洞察卡"而非重复生成
- 统计洞察卡生成后的用户打开率，作为质量反馈

---

## 5. 今日报告优化

### 5.1 当前状态

- **规则版报告**（`GET /radar/daily-report`）：可用，无需 LLM
- **LLM 版核心报告**（`POST /radar/today/daily-report`）：默认关闭

### 5.2 下一阶段目标

**规则版报告输出有价值**：

1. 确认规则版报告能展示当日最重要的 10-15 条内容
2. 确认分类正确（今日重点 / Agent 工作流 / RAG 知识库等）
3. 确认摘要内容可读（不出现 raw title 或乱码）

### 5.3 LLM 版核心报告策略

- 保持 `DAILY_REPORT_ENABLED=false` 默认关闭
- 用户确认规则版有用后再开启 LLM 版
- 开启时提示预估成本

---

## 6. 正文抓取确认

### 6.1 当前状态

- 支持"获取 HTML 正文"按钮
- 但没有显示抓取成功率统计
- 用户不知道哪些来源抓取失败

### 6.2 下一阶段目标

**确认抓取成功率**：

1. 在"运行状态"区显示最近抓取的成功率
2. 失败的来源显示具体错误（403 / 404 / timeout 等）
3. 对高频失败的来源给出提示
