# V0.7 真实高价值来源覆盖验收

## 目标

验证 AI Frontier Radar 不只依赖 Hugging Face，而是能从 Anthropic、DeepMind、Mistral 等高价值来源发现真实文章。

---

## 默认验收来源

- `anthropic_news` — Anthropic 新闻
- `deepmind_blog` — Google DeepMind 博客
- `mistral_ai_news` — Mistral AI 新闻

---

## 验收命令

```bash
python scripts/acceptance_real_source_coverage.py --isolated-db --repeat 2 --timeout 15
```

指定来源：

```bash
python scripts/acceptance_real_source_coverage.py --source-key anthropic_news --source-key deepmind_blog --source-key mistral_ai_news --isolated-db --repeat 2 --timeout 15
```

---

## 质量检查命令

```bash
python scripts/check_source_item_quality.py --source-key anthropic_news
python scripts/check_source_item_quality.py --source-key deepmind_blog
python scripts/check_source_item_quality.py --source-key mistral_ai_news
```

---

## 如何看结果

### acceptance_real_source_coverage.py 输出

每个来源返回：

```
anthropic_news  [html_index]
  status: success
  items_found=5, new=3, updated=2, failed=0
  error: (none)
  article_like=5, expected_content=5, suspected_listing=0, suspected_off_topic=0
  Top expected SourceItems:
    - Anthropic Blog Post Title
      https://anthropic.com/news/post-title
```

### check_source_item_quality.py 输出

每个 source_key 返回：

```
  total items: 10
  empty_title_count: 0
  suspected_listing_count: 0
  expected_content_count: 8
  suspected_off_topic_count: 2
  duplicate_url_count: 0
```

### 字段说明

- `article_like_count`：不是列表页的 URL 数
- `expected_content_count`：符合该 source_key 预期内容路径的 URL 数
- `suspected_off_topic_count`：不是列表页，但不符合该来源目标内容类型的 URL 数
- `suspected_listing_count`：列表页/分页页 URL 数

---

## 通过标准

1. 脚本整体不崩溃
2. 至少 2 个来源完成探测流程（status = success 或 partial_failed）
3. 至少 2 个来源发现 expected_content SourceItem
4. repeat=2 时不出现唯一约束错误
5. 失败来源必须有 `last_error_message` 或 `FetchRun.error_message`

---

## V0.7.1 加强来源质量判断

V0.7.1 加强了来源质量判断：不仅判断是否列表页，还判断是否符合来源预期内容类型。

- DeepMind `/models/` 页面会被标记为 `suspected_off_topic`，不再算作 `deepmind_blog` 的高质量博客文章
- DeepMind `/discover/blog/` 和 `/discover/research/` 路径的文章会被标记为 `expected_content`
- Anthropic `/news/` 路径的文章会被标记为 `expected_content`
- Mistral `/news/` 路径的文章会被标记为 `expected_content`

## 常见问题

| 问题 | 原因 | 处理 |
|------|------|------|
| HTTP 403 | 官网反爬 | 记录原因，不绕过 |
| 结构变化导致候选链接为空 | HTML 结构更新 | 标记 partial_failed，更新探测规则 |
| 网络超时 | 网络不稳定 | 增加 timeout 或重试 |
| 发现的是列表页 URL | 过滤规则不够严格 | suspected_listing_count > 0 时输出 warning |
| 发现的是 /models/ 页 | DeepMind 产品页不是博客 | suspected_off_topic_count > 0，改进过滤规则 |
| arXiv RSS 返回 0 entries | 源本身无更新 | 正常，记录即可 |

## 本轮不做什么

- 不批量探测全部 15 个来源
- 不做后台调度
- 不做浏览器自动化（Playwright）
- 不调用 LLM 判断 URL
