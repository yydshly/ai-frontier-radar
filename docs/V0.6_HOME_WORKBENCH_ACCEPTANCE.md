# V0.6 首页个人工作台验收

## 目标

验证用户打开首页后，能知道当前有哪些资料要处理，哪些卡片要回看，哪些行动任务可导出。

## 产品闭环

```
首页
→ 待编译资料
→ 中文洞察卡
→ 用户判断
→ 转成行动
→ Markdown 任务导出
```

## 前置条件

- 已安装依赖
- 数据库中有数据（可以用 acceptance 脚本自动创建）
- 不需要 LLM API Key

---

## 手动验收步骤

### 1. 启动服务

```bash
uvicorn app.main:app --reload --port 8779
```

### 2. 打开首页 `/`

确认首页不是单纯的 URL 提交页，而是工作台形态。

### 3. 查看统计卡片

确认显示 4 个主卡片：

- 待编译资料 → 点击跳转 `/source-items`
- 未处理卡片 → 点击跳转 `/cards?decision=unhandled`
- 值得关注 → 点击跳转 `/cards?decision=worth_attention`
- 转成行动 → 点击跳转 `/cards?decision=to_action`

### 4. 查看下一步建议

确认根据当前数据状态，显示相应的建议文案。

### 5. 点击待编译资料

跳转到 `/source-items`，确认有数据。

### 6. 点击未处理卡片

跳转到 `/cards?decision=unhandled`，确认有数据。

### 7. 点击转成行动

跳转到 `/cards?decision=to_action`，确认有数据。

### 8. 从 to_action 卡片进入导出任务

点击某张 to_action 卡片的"导出任务"，跳转到 Markdown 预览页。

### 9. 确认保留功能

- 手动 URL 编译入口仍在
- 精选来源展示仍在

## 验收标准

- [ ] 首页是工作台形态，不是单纯 URL 提交页
- [ ] 用户能看到统计卡片（待编译资料、未处理卡片、值得关注、转成行动）
- [ ] 统计卡片可点击跳转
- [ ] 下一步建议区根据数据状态动态显示
- [ ] 快捷入口存在（收件箱、洞察卡、行动任务、信息来源）
- [ ] 最近待编译资料区存在
- [ ] 最近中文洞察卡区存在（含 to_action 导出入口）
- [ ] 手动 URL 编译入口保留
- [ ] 精选来源保留
- [ ] 数据库为空时首页仍能正常渲染
