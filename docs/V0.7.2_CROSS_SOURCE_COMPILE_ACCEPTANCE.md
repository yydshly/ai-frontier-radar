# V0.7.2 跨来源单条编译验收

## 目标

验证 Anthropic / Mistral / DeepMind 的 expected SourceItem 能否编译成中文 InsightCard。

## 为什么要做

发现 URL 不等于产品可用。只有能抓正文、生成中文 InsightCard，来源才真正有价值。

V0.7.1 验证了 URL 质量（不是列表页、符合来源预期内容类型）。V0.7.2 验证这些高质量 URL 能否进入正文抓取和 LLM 编译链路。

---

## 验收来源

默认：
- `anthropic_news`
- `mistral_ai_news`

可选：
- `deepmind_blog`

---

## 验收命令

### Mock 模式（不调用真实 LLM，验证链路）

```bash
python scripts/acceptance_cross_source_compile.py --isolated-db --mock-llm
```

### 真实模式（验证端到端）

```bash
# 单来源
python scripts/acceptance_cross_source_compile.py --isolated-db --source-key anthropic_news

# 多来源
python scripts/acceptance_cross_source_compile.py --isolated-db --source-key anthropic_news --source-key mistral_ai_news

# 指定超时
python scripts/acceptance_cross_source_compile.py --isolated-db --timeout 20
```

### 保留 DB 方便排查

```bash
python scripts/acceptance_cross_source_compile.py --isolated-db --keep-db --mock-llm
```

## 如何看结果

脚本输出每个来源的编译摘要：

```
anthropic_news
  selected_url: https://www.anthropic.com/news/claude-opus-4-8
  compile_status: completed
  insight_card_id: 42
  quality:
    summary_present: True
    key_points_count: 3
    technical_insights_count: 2
    product_opportunities_count: 1
    action_items_count: 2
    relevance_score_present: True
    passed_minimum_quality: True
```

### inspect_insight_card_quality() 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `summary_present` | bool | summary_zh 非空 |
| `key_points_count` | int | key_points_zh 数组长度 |
| `technical_insights_count` | int | technical_insights_zh 数组长度 |
| `product_opportunities_count` | int | product_opportunities_zh 数组长度 |
| `risks_count` | int | risks_zh 数组长度 |
| `action_items_count` | int | action_items_zh 数组长度 |
| `relevance_score_present` | bool | relevance_score > 0 |
| `related_directions_count` | int | related_user_directions 数组长度 |
| `passed_minimum_quality` | bool | 通过最低质量标准 |
| `warnings` | list[str] | 质量问题列表 |

### 最低质量标准

`passed_minimum_quality=True` 当且仅当：

1. `summary_zh` 非空
2. 至少 2 个结构化字段（key_points / technical_insights / product_opportunities / action_items）非空
3. `relevance_score` > 0

---

## 通过标准

1. Mock 模式必须通过（所有来源 quality 检查通过）
2. 真实模式至少 1 个来源成功生成 `completed` InsightCard
3. 失败来源必须有明确 `error_message`
4. repeat=1，无幂等性要求

---

## 常见问题

| 问题 | 原因 | 处理 |
|------|------|------|
| API Key 缺失 | MINIMAX_API_KEY 未配置 | 记录原因，真实模式失败是可接受的 |
| LLM 调用失败 | 模型服务不可用 | 记录 error_message |
| 正文抓取失败 | HTTP 403/404 | 记录 error_message |
| JSON 解析失败 | LLM 输出格式问题 | 记录 error_message |
| 质量不达标 | 字段缺失或内容不足 | passed_minimum_quality=False |

---

## 本轮不做什么

- 不批量编译全部 SourceItem
- 不做后台队列
- 不做自动调度
- 不调用 LLM 批量分析
