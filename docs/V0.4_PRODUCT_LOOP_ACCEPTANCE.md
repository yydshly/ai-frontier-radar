# V0.4 产品目标闭环人工验收

## 目标

验证中文用户能走完完整的「英文资料 → 中文 InsightCard → 个人判断」闭环。

本轮要确认的不是系统能编译卡片（V0.3.2 已经做完了），而是：

```text
用户看完卡片后，能否明确表达这篇内容对自己「值不值得关注 / 是不是和我的方向有关 / 要不要转成行动」。
```

完成后，用户在 `/cards` 列表里就能一眼看到每张卡片的「处理状态」，而不是一堆未读的 completed 卡片。

---

## 产品闭环

```text
待编译资料收件箱（/source-items?source_key=huggingface_blog）
→ 进入详情（/source-items/{id}）
→ 编译为 InsightCard
→ 阅读中文卡片（/cards/{id}）
→ 在「看完后的判断」区选择一个决策：
   - 值得关注
   - 与我有关
   - 稍后再看
   - 暂时忽略
   - 转成行动
→ 回到 /cards 列表，确认处理状态显示正确
```

---

## 前置条件

- 已安装依赖
- 数据库文件可写
- 不需要 LLM API Key（可以用已有 completed 卡片验收，或用 isolated DB 跑自动验收）

---

## 步骤

### 1. 探测 huggingface_blog（如果你还没有 SourceItem）

```powershell
python scripts\acceptance_probe_sources.py --repeat 2 --timeout 15 --html-source huggingface_blog
```

确认 `items_found > 0` 且 Top discovered 全部是 `/blog/{slug}` 形式。

### 2. 启动 web 服务

```powershell
uvicorn app.main:app --reload --port 8779
```

### 3. 打开待编译资料收件箱

```text
http://127.0.0.1:8779/source-items?source_key=huggingface_blog
```

### 4. 选择一条文章

挑选标准：

- URL 是 `/blog/{slug}`（不是 `/blog?p=2`、不是 `/blog`）
- 标题像具体文章

点击「推荐操作」列里的「进入详情并编译」按钮。

### 5. 编译为 InsightCard

详情页底部点「编译为 InsightCard」按钮。

- 如果你已配置 LLM API Key：会生成中文摘要、关键事实、技术洞察、产品机会、风险、行动建议
- 如果没有 API Key：会生成 failed 卡片（仍能继续验证决策流程）

页面会跳回 `/source-items/{id}`，状态变为 `compiled` 或 `failed`，出现「查看中文卡片」或「查看失败原因 / 重试」按钮。

### 6. 打开 InsightCard 详情

点「查看中文卡片」链接，进入 `/cards/{id}`。

你应该看到：

- 顶部 `🎴 InsightCard 详情` 标题
- `📄 原文信息`（标题、URL、类型、作者、发布日期等）
- `📝 中文摘要`（completed 卡片才有）
- `🔑 关键事实`
- `💡 技术洞察`
- `🚀 产品机会`
- `⚠️ 风险`
- `🎯 相关性判断`（分数 + 匹配方向 + 理由）
- `✅ 行动建议`
- `🧭 看完后的判断` ← **本轮新增**

### 7. 在「看完后的判断」区选一个决策

应该看到：

```text
当前判断：未处理
```

下方有 5 个单选：

```text
○ 值得关注
○ 与我有关
○ 稍后再看
○ 暂时忽略
○ 转成行动
```

可选项：备注文本框（写「为什么」「和哪个项目有关」「下一步想做什么」）。

选择一个 + 可选填备注 → 点「保存判断」。

页面会跳回 `/cards/{id}`，现在显示「当前判断：xxx」。

### 8. 回到 `/cards` 列表确认处理状态

打开 `http://127.0.0.1:8779/cards`。

列表里你应该看到一行带颜色徽章：

```text
| ID | 标题 | URL | 状态 | 处理状态 | 相关性 | 创建时间 | 操作 |
```

「处理状态」列显示你刚才选的判断中文标签。

## 验收标准

完成上述步骤后，请确认：

```text
[ ] /cards/{id} 页面底部能看到「🧭 看完后的判断」区块
[ ] 区块里能看到 5 个单选：值得关注 / 与我有关 / 稍后再看 / 暂时忽略 / 转成行动
[ ] 新卡片默认显示「当前判断：未处理」
[ ] 保存判断后页面跳回 /cards/{id}，「当前判断」更新为新选择
[ ] 当前已选择的 radio 显示为 checked
[ ] 之前填过的 note 文本框会回显
[ ] 再次保存会更新同一行（不会报错说已经存在）
[ ] /cards 列表能看到「处理状态」列
[ ] /cards 列表的处理状态列显示你保存的中文判断
[ ] 用户即使英语能力有限，也能明确说出每张卡片「值不值得关注」
```

任意一项打不到，说明 V0.4 产品闭环没有真正达成。

## 验收命令（自动化）

如果你不想手动点 UI，可以用自动验收脚本：

```powershell
python scripts\acceptance_card_decision.py --isolated-db
```

脚本会：

1. 在 isolated DB 中创建一张 completed InsightCard
2. 打开 `/cards/{id}` 验证决策区显示
3. POST `decision=worth_attention` 验证创建
4. POST `decision=to_action + note` 验证 update（不 insert）
5. 打开 `/cards` 验证处理状态显示「转成行动」
6. 输出 `ACCEPTANCE PASSED`

不会联网、不会调用 LLM、不会污染你本地数据。

## 当前范围

V0.4 仍是**单用户本地 MVP**：

```text
✓ 一张卡只有一个当前决策
✓ 决策是用户自己的判断（不是模型判断）
✓ 不做多用户
✓ 不做推荐算法
✓ 不做自动评分
✓ 不做批量处理
✓ 不做任务管理
```

## 下一步

如果 V0.4 通过，下一轮可以考虑：

- 按处理状态过滤 `/cards`（如只看「转成行动」或「与我有关」）
- 在 `/source-items` 中显示已编译卡片的处理状态
- 将「转成行动」的卡片导出为 Markdown 任务

但本轮先完成「看完后做判断」这件事本身。
