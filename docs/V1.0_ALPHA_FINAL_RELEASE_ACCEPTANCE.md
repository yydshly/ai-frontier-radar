## 已知限制

详见 [docs/KNOWN_LIMITATIONS.md](KNOWN_LIMITATIONS.md)，重点包括：

- 单用户本地使用，无多用户/登录注册
- SQLite 本地数据库，无高并发支持
- 无数据库迁移系统，模型变更后需重建 DB
- 真实网页抓取可能失败（403/结构变化/RSS 空）
- mock 验收只验证链路，真实 LLM 输出需人工判断
- GitHub Actions 只跑基础 CI，不跑真实网络/LLM
- Windows 下 isolated DB 删除有文件锁 warning（不影响 Linux CI）

## V1.0-alpha.8.2 输入识别与策略路由（V1.0-alpha.8.2）

人工浏览器验收发现列表页（如 DeepMind `/blog/page/3/`）可能被错误编译为 InsightCard。
已新增 URL 类型识别模块，阻止列表页、分页页、标签页、feed 直接进入编译：

| 问题 | 修复内容 | 修改文件 |
|------|---------|---------|
| 列表页被错误编译 | 新增 `app/intake/` 模块，包含 `PageType`、`RecommendedStrategy`、`IntakeDecision` 数据模型和 `classify_url_by_pattern` 规则分类器 | `app/intake/__init__.py`, `app/intake/models.py`, `app/intake/url_classifier.py` |
| 手动 URL 编译未拦截 | `POST /compile` 接入分类 gate，blocked 时创建 failed card | `app/main.py` |
| SourceItem 编译未拦截 | `POST /source-items/{id}/compile` 接入分类 gate，blocked 时标记 SourceItem 为 failed | `app/main.py` |
| 阻止后页面提示不清 | SourceItem 详情页对 `[intake:blocked]` 错误显示专门引导文案 | `source_item_detail.html` |
| 无策略说明文档 | 新增 `docs/INPUT_CLASSIFICATION_AND_SUMMARY_STRATEGY.md` | — |

**URL 类型覆盖：**

| URL 示例 | 类型 | 策略 |
|----------|------|------|
| `/blog/page/3/` | `pagination` | discovery_only |
| `/blog/` | `listing` | discovery_only |
| `?tag=agents` | `tag_or_category` | discovery_only |
| `/feed.xml` | `feed` | discovery_only |
| `.pdf` | `pdf` | compile |
| `/discover/blog/sima-2-agent/` | `article` | compile |
