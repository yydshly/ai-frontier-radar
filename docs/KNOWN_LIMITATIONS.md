# V1.0-alpha Known Limitations

## 当前定位

V1.0-alpha 是**本地个人演示版**，不是公开生产版本。
核心目标是验证"英文 AI 前沿资料 → 中文洞察 → 可执行任务"这条主链路在本地可运行、可演示。

---

## 已知限制

### 1. 单用户本地使用

**当前没有：**
- 登录注册
- 多用户
- 权限管理
- 用户数据隔离

**影响：** 只能一个人本地使用，不能协作。

---

### 2. SQLite 本地数据库

**当前使用 SQLite（文件数据库）。**

**影响：**
- 适合本地演示和开发
- 不适合高并发（多人同时使用）
- 不适合大数据量（ > 10 万条记录）

**如果需要：** 考虑迁移到 PostgreSQL（未来选项，非本次 Release 范围）。

---

### 3. 无数据库迁移系统

**当前使用 `Base.metadata.create_all()` 初始化数据库。**

**影响：**
- 模型（表结构）变更后，旧的 SQLite 文件不能自动迁移
- 需要手动删除旧 DB 或重建

**当前 workaround：** 开发阶段删除 `data/` 下的 `.db` 文件重建。

---

### 4. 真实网页抓取不稳定

**部分来源可能遇到：**
- HTTP 403 / 429（来源站点反爬）
- 页面结构变化（CSS 选择器失效）
- RSS 返回空数据
- 候选链接质量参差不齐（过滤逻辑仍有误判可能）

**影响：** 某些来源可能在首次探测时失败，需要重试或人工介入。

---

### 5. 真实 LLM 输出质量依赖人工判断

**mock 验收只验证链路通不通。**

**真实 LLM 输出仍需要：**
- 保真检查（英文摘要是否忠实原文）
- 中文解释检查（是否曲解原意）
- 用户最终人工判断

**当前产品中：** 没有自动评估 LLM 输出质量的功能，纯靠人工判断。

---

### 6. GitHub Actions 只跑基础 CI

**CI 不跑：**
- 真实网络探测
- 真实 LLM 调用
- 需要 API Key 的验收
- 大规模抓取
- 后台任务

**影响：** CI 通过不代表真实网络来源可访问或 LLM 输出质量好。

---

### 7. Windows 文件锁导致 isolated DB 删除 warning

**Windows 下运行 health_check 时可能出现：**

```
[WARN] Could not delete isolated DB (Windows file lock)
```

**原因：** Windows 对打开的 SQLite 文件加锁，Python 进程退出后有时延迟释放。

**影响：** 仅是 warning，不影响 CI 结果，Linux CI 不受影响。

---

### 8. SourceItem 编译逻辑仍在路由层

**当前 `POST /source-items/{id}/compile` 和 `POST /compile` 的编译逻辑直接在 `app/main.py` 路由中。**

**影响：**
- 无法批量编译（只能一次一个 URL）
- 无法作为后台任务运行
- 难以被其他模块复用

**建议（V1.0-beta 前）：** 抽取到 `app/services/source_item_compiler.py`。

---

### 9. 产品形态仍是本地工作台

**当前还不是：**
- SaaS（云端服务）
- 浏览器插件
- 移动 App
- 自动周报推送系统
- 多源自动追踪

**影响：** 需要用户手动触发编译，不能自动追踪前沿更新。

---

### 10. demo 数据是静态 mock

**demo 数据不是从真实来源抓取，内容是预设的。**

**影响：** 演示时不能体现真实来源探测和 LLM 编译效果，只能展示后续的"中英双语报告 → 行动任务"链路。

---

## 下一阶段建议（V1.0-beta 前优先）

1. **SourceItem 编译 service 化** — 抽取编译逻辑，支持批量和后台任务
2. **数据库迁移策略** — 引入 Alembic 或类似工具
3. **真实来源质量策略** — 改善 HTML 索引过滤、RSS 容错
4. **页面体验打磨** — 加载状态、错误提示、空状态 UI
5. **更清晰的错误恢复机制** — 重试逻辑、失败原因展示

---

## 不在 V1.0-alpha 范围内的能力

以下能力在 V1.0-alpha 中**刻意不做**，属于未来版本考虑：

| 能力 | 原因 |
|------|------|
| 多用户/登录注册 | MVP 阶段聚焦单用户主链路 |
| 后台批量编译 | 先跑通单条，再考虑批量 |
| 向量数据库/语义搜索 | 需要先有真实数据积累 |
| 知识图谱 | 需要先有实体和关系数据 |
| SaaS 部署 | 需要多用户和数据隔离 |
| 自动周报 | 需要先有真实消费数据 |
