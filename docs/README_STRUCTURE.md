# README 结构说明

## 目标

README 是项目入口，不是完整开发流水账。

## 推荐阅读顺序

1. 先看 5 分钟本地演示
2. 再看产品主流程
3. 再看当前核心能力
4. 需要开发时看 `docs/IMPLEMENTATION_GUIDE.md`
5. 需要理解架构时看 `docs/ARCHITECTURE_OVERVIEW.md`

## README 中应该放什么

- 项目定位
- Quickstart
- 主流程
- 常用命令
- 文档入口
- 当前边界
- 版本路线摘要

## README 中不应该过度堆什么

- 每一轮详细验收日志
- 长篇实现细节
- 大量历史任务描述
- 过长的排查记录

## 历史版本记录处理原则

主 README 只保留摘要，详细验收文档放 `docs/`。
