## 修复详情

### 1. create_demo_data.py --reset-demo 删除逻辑

**问题**：原删除逻辑使用 `InsightCard.source_url == SourceItem.url`（类属性），导致删除失败。

**修复**：
- 收集 source_key 下所有 SourceItem
- 优先用 `insight_card_id`，同时也用 `url` 匹配 `source_url`
- 按正确顺序删除：`CardDecision` → `InsightCardBilingualReport` → `InsightCard` → `SourceItem` → `Source`

### 2. demo 数据半残留重建逻辑

**问题**：如果 source_key 存在但数据不完整，继续创建会触发唯一约束。

**修复**：source_key 存在但 demo 数据不完整时，输出 warning，调用 `_delete_demo_data` 清理后重新创建。

### 3. health_check.py quick/full 行为

**问题**：默认 quick 模式也跑 smoke_test，不符合 quick 设计。

**修复**：
- `python scripts/health_check.py`（默认 quick）：不跑 smoke_test，不跑 acceptance
- `python scripts/health_check.py --full`：跑 smoke_test + acceptance
- `python scripts/health_check.py --full --skip-smoke`：不跑 smoke_test，但跑 acceptance

### 4. acceptance_ci_local.py env 传递

**问题**：`run_command()` 没有把 `env` 传给 `subprocess.run()`。

**修复**：
- `run_command()` 增加 `env` 参数
- `subprocess.run()` 调用增加 `env=env`
- 输出中显示 `DATABASE_URL=sqlite:///data/ci.db`

### 5. acceptance_ui_links.py 占位链接假阳性

**问题**：发现 `/cards/{id}/export-report` 占位链接只 WARN，不 FAIL。

**修复**：
- 改用 `assert not matches` 直接失败
- 增加真实链接存在性检查（`/cards/{id}/export-report` 和 `/cards/{id}/export-markdown`）
- 增加下载路由验证：`/cards/{id}/export-report/download` 返回 200 且 Content-Disposition 包含 `insightcard-{id}-report.md`

### 6. FastAPI app version

**问题**：version 仍是 0.1.0，与 V1.0-alpha 阶段不一致。

**修复**：
- 新增 `app/version.py`，内容为 `APP_VERSION = "1.0-alpha"`
- `app/main.py` 导入 `APP_VERSION` 并用于 FastAPI app
