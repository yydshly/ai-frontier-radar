# V1.0-alpha 可演示主流程

## 目标

把现有能力整理成一条稳定可演示路径，让用户和后续执行模型都能清楚知道如何从"发现资料"走到"完整报告导出"。

## 主流程概览

```
首页工作台
→ SourceItem 收件箱（第 2 步）
→ SourceItem 详情
→ InsightCard（第 4～6 步）
→ BilingualReport
→ CardDecision
→ Full Markdown Report
→ Action Markdown Task
```

## 详细演示步骤

### 第 1 步：打开首页工作台

**入口**：`/`

**用户看到**：
- 🤖 AI Frontier Radar 标题
- 📊 工作台概览统计卡片（待编译资料、未处理卡片、值得关注、转成行动）
- 🚦 **推荐主流程**区块（新增）
- 💡 下一步建议
- ⚡ 快捷入口

**用户点击**：`待编译资料收件箱` 或 `进入待编译资料`

---

### 第 2 步：进入 SourceItem 收件箱

**入口**：`/source-items`

**页面提示**（新增）：
```
🚦 你现在在主流程第 2 步
选择一条英文资料，编译为中文 InsightCard。

优先选择标题像具体文章的条目，例如 /news/...、/blog/...，
不要选择列表页或分页页。
```

**用户操作**：
- 查看待编译资料列表
- 优先选择标题像具体文章的条目（如 `/blog/xxx`、`/news/xxx`）
- 点击「进入详情并编译」进入详情页

---

### 第 3 步：SourceItem 详情页

**入口**：`/source-items/{id}`

**下一步提示**（根据状态显示，新增）：

| 状态 | 提示文案 |
|------|---------|
| discovered | 🚦 下一步：点击「编译为 InsightCard」，生成中文洞察卡。 |
| compiled | 🚦 下一步：打开关联 InsightCard，生成中英双语报告并做判断。 |
| failed | 🚦 下一步：查看失败原因，可以稍后重试或换一条资料。 |

**用户操作**：
- 点击「编译为 InsightCard」按钮提交编译
- 或点击「查看中文卡片」打开已编译的卡片

---

### 第 4 步：打开 InsightCard

**入口**：`/cards/{id}`

**主流程提示**（新增）：
```
🚦 你现在在主流程第 4～6 步
阅读 InsightCard，生成中英双语核心理解，并做出你的判断。
```

**双语报告区提示**（新增）：
```
💡 如果你担心中文解说曲解原文，建议先看 English Core Summary
和 Original Key Claims，再看中文解说。
```

**用户操作**：
- 阅读中文摘要、关键事实、技术洞察、产品机会、风险、行动建议
- 决定是否需要生成中英双语报告

---

### 第 5 步：生成中英双语报告

**入口**：`/cards/{id}` → 「🌐 生成中英双语报告」按钮

**用户看到**：
- English Core Summary（英文核心摘要）
- Original Key Claims（原文主张）
- Key Evidence Points（证据点）
- Key Terms EN-ZH（术语中英对照）
- 中文解说
- 保真提示
- 解读边界

---

### 第 6 步：做出用户判断

**入口**：`/cards/{id}`

**判断区提示**（新增）：
```
💡 判断不是模型做的，而是你对这篇资料的个人处理结果。
```

**决策选项**：
- 值得关注（worth_attention）
- 与我有关（related_to_me）
- 稍后再看（read_later）
- 暂时忽略（ignore）
- 转成行动（to_action）

**用户操作**：
- 选择一个判断
- 可选添加备注
- 点击「保存判断」

---

### 第 7 步：导出完整 Markdown 报告

**入口**：`/cards/{id}` → 「📄 导出完整 Markdown 报告」

或直接访问：`/cards/{id}/export-report`

**报告内容**：
- 英文核心摘要（English Core Summary）
- 原文主张（Original Key Claims）
- 证据点（Key Evidence Points）
- 术语中英对照（Key Terms EN-ZH）
- 中文解说（Chinese Explanation）
- 中文摘要
- 关键事实
- 技术洞察
- 产品机会
- 风险
- 行动建议
- 保真提示
- 解读边界
- 用户判断
- 后续追问问题

**适用场景**：长期保存和复盘

---

### 第 8 步：（可选）导出行动任务

**入口**：`/cards/{id}` → 「📤 导出为 Markdown 任务」或「🔍 预览 Markdown 任务草稿」

或直接访问：`/cards/{id}/export-markdown`

**适用场景**：继续交给 AI 拆解执行

---

## 页面入口汇总

| 页面 | 路径 | 主流程位置 |
|------|------|-----------|
| 首页工作台 | `/` | 第 1 步 |
| SourceItem 收件箱 | `/source-items` | 第 2 步 |
| SourceItem 详情 | `/source-items/{id}` | 第 3 步 |
| InsightCard 详情 | `/cards/{id}` | 第 4～6 步 |
| 完整报告导出 | `/cards/{id}/export-report` | 第 7 步 |
| 行动任务导出 | `/cards/{id}/export-markdown` | 第 8 步 |

## 当前边界

本版本**不做**以下功能：

- 批量编译
- 后台任务
- 多用户
- 公开 SaaS
- PDF / DOCX 导出
- Notion / 飞书同步
- 移动端适配
- 浏览器插件
