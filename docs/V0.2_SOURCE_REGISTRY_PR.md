# V0.2 Source Registry 与来源探测链路

## 背景

V0.1 验证了最小闭环：输入 URL → 获取正文 → LLM 生成中文 InsightCard → Web 查看。但 V0.1 依赖人工发现 URL，无法主动追踪 AI 前沿来源。

V0.2 在 V0.1 基础上新增了**来源配置 → 探测 → 发现 → 手动编译**的完整闭环，实现半自动化的 AI 前沿知识挖掘。

## 本次变更

本次变更包含以下模块的完整实现：

| 模块 | 描述 |
|------|------|
| 配置来源 | YAML 配置加载，Source ORM 模型，DB 同步 |
| 来源列表页 | `/sources` 页面展示所有配置来源 |
| RSS 探测 | `probe_rss_source` / `probe_rss_sources.py`，支持 feed 探测 |
| HTML Index 探测 | `probe_html_index_source` / `probe_html_index_sources.py`，支持博客/新闻索引页探测 |
| 发现条目列表 | `/source-items` 页面，支持按来源/状态/关键词筛选 |
| 发现条目详情 | `/source-items/{item_id}` 页面，展示条目完整信息 |
| 单条手动编译 | `POST /source-items/{item_id}/compile`，复用现有 `compile_url()`，更新 SourceItem 状态 |

## 不在本次范围

- 批量编译
- 后台调度 / 定时任务
- 探测后自动编译
- 多用户 / 登录注册
- Alembic 数据库迁移
- 复杂正文抓取质量治理
- 完整推荐系统

## 已知风险

1. **事务边界**：`probe_rss_source` / `probe_html_index_source` 内部存在 `commit`，批量任务前建议统一事务边界
2. **同步编译阻塞**：SourceItem 手动编译是同步请求，长耗时 URL 或 LLM 调用会阻塞请求
3. **无中间状态**：尚未实现编译中间状态（如 `compiling`），用户需等待才知道是否成功
4. **无 Alembic**：当前使用 `Base.metadata.create_all`，暂不支持生产级迁移
5. **真实探测未入测试**：RSS / HTML 探测脚本真实网络结果尚未纳入 smoke_test
