📌 支持的内容类型

✅ 当前支持

  • 英文 AI 前沿文章网页
  • 英文研究博客 / 公司公告
  • 可直接访问的 PDF 报告

❌ 暂不支持

  • 需要登录的网站
  • Twitter / X
  • YouTube 视频
  • 扫描版图片 PDF
  • 复杂反爬网站

🔍 系统如何处理 URL

1

抓取网页或 PDF

系统使用 HTTP 请求获取原始内容，并根据 Content-Type 判断是 HTML 还是 PDF。

2

提取正文

HTML 页面优先使用 trafilatura 提取正文，失败时使用 BeautifulSoup fallback。
PDF 文件使用 pypdf 提取文本。

3

清洗内容

移除多余空白、导航残留、页脚噪音，并限制最大正文长度（60,000 字符）。

4

去重

根据清洗后的正文计算 SHA256 hash，避免同一 URL 和同一内容重复生成卡片。

5

大模型分析

调用当前配置的 LLM Profile，将英文资料编译成中文 InsightCard，包含摘要、关键事实、技术洞察、产品机会、风险和相关性判断。

6

保存和查看

生成结果保存到 SQLite，可在 InsightCard 列表和详情页查看。

⚙️ 当前 LLM Profile 配置

当前默认 Profile：minimax_m27_highspeed_anthropic

默认模型：MiniMax-M2.7-highspeed

配置文件：config/llm_profiles.yaml（可通过 .env 中的 LLM_PROFILE 切换）

切换模型示例：

  • LLM_PROFILE=minimax_m27_highspeed_anthropic — 默认高速模型
  • LLM_PROFILE=minimax_m3_anthropic — 更高质量模型
  • LLM_PROFILE=openai_compatible_default — OpenAI-compatible fallback

🔎 失败排查说明

如果处理失败，系统不会直接丢失任务，而是会创建 failed card。

你可以在详情页看到失败原因，例如：

  • URL 无法访问
  • 正文提取失败
  • PDF 无法解析
  • API Key 未配置
  • 大模型返回的 JSON 无法解析

访问 InsightCard 列表 查看所有卡片（包括 failed）。
